心脏病数据处理作业展示

探索临床医学公开数据库，通过数据挖掘识别心脏病风险因素

作者：翁浩然 | 学号：2510108305

开始探索

数据介绍

本项目使用的数据集来自 Kaggle 平台，包含 10,000 条记录和 21 个特征。该数据集涵盖了与心脏病相关的各种健康指标和风险因素，旨在研究与心脏病相关的趋势并识别关键风险因素。

  • 数据集名称： Heart Disease Dataset (Kaggle)
  • 样本量： 10,000 条临床记录
  • 特征数量： 20 个预测因子 + 1 个目标变量
  • 核心指标： 年龄、血压、胆固醇、BMI、糖尿病、吸烟习惯、睡眠时长等

* 注：由于数据集本身特征相关性较低，模型建立过程中遇到了 AUC 接近 0.5 的挑战。

数据读取

使用 Python 的 Pandas 库加载 CSV 数据文件，并初步查看数据维度与结构。

import pandas as pd
# 读取数据集
df = pd.read_csv('heart_disease.csv')
# 查看数据维度 (10000, 21)
print(df.shape)
# 展示前五行数据
df.head()

通过 df.info() 发现数据集中存在少量缺失值，涉及年龄、血压、胆固醇等多个字段，需要进一步预处理。

数据预处理

缺失值处理

针对数值型变量（如年龄、BMI），采用中位数进行填充；针对分类型变量（如性别、吸烟），采用众数进行填充。

特征工程

将性别、吸烟等二分类变量映射为 0/1；对锻炼习惯、压力水平等定序变量进行等级编码（0, 1, 2）。

统计分析

为了探究指标与心脏病的关系，我们将样本分为健康组与患病组，并进行了以下检验：

Mann-Whitney 连续变量检验
Chi-Square 分类变量检验
Heatmap 特征相关性分析

分析结果显示，大部分特征与心脏病状态的相关系数均小于 0.1，这解释了后续模型预测性能受限的原因。

预测模型建立

本项目尝试了多种机器学习算法来构建心脏病预测模型：

逻辑回归 (Logistic Regression)： 作为基准模型进行线性分类。
随机森林 (Random Forest)： 利用集成学习处理非线性关系并评估特征重要性。
划分比例： 采用 train_test_split 将数据按 8:2 比例划分为训练集与测试集。

预测模型评估与可视化

通过混淆矩阵和 ROC 曲线对模型性能进行全面评估。

评估指标

主要关注 AUC (Area Under Curve)、准确率 (Accuracy) 以及针对医学场景的召回率 (Recall)。
